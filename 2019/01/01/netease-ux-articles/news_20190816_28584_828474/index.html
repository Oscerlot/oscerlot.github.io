Flink is a stream processing framework. In stream processing, data can usually be regarded as an infinite-capacity pipeline. Every time a message comes in, it is processed. This is a different concept from batch processing. Batch processing usually stores data in a file system and then processes the data.

From this, it can be seen that compared to the batch processing method, the stream processing calculation approach has lower latency and is more suitable for scenarios such as real-time calculation.

So how do we count data in this stream processing framework? In Flink, a variety of windows are provided to help us implement the calculation and statistics of stream data. This article will briefly introduce the concept of windows in Flink.

Consider the following scenario: cars are constantly passing through a checkpoint from the same direction. We want to count the total number of cars passing through this checkpoint within a period of time. Since the cars are constantly coming, we cannot wait for all the cars to pass through this checkpoint before counting and summing.

So we need to change our thinking and try an accumulation strategy.
