Introduction to Recommender System Technology
1 Jan, 2019

Machine Translated from Chinese to English
Original article written by ThunderFire UX (NetEase)

I believe everyone has had the following experiences:

You finish a day’s work and open a video website, and the banner shows exactly the show you want to watch.

Having just received your salary at the beginning of the month, you can’t help but open a shopping app, and the scrolling ads on the homepage remind you that it’s time to splurge.

Personalized recommendation systems that cater to each individual bring “understanding” and precise recommendations. Today, we will briefly introduce the recommendation technology behind them.

Problem Definition

Recommendation problem: Implement a utility function that automatically predicts a user’s degree of liking for a certain item.

Algorithm Classification

The industry and academia have proposed many algorithms for the implementation of recommender systems. Some typical algorithms are as follows:

Collaborative Filtering Recommendation Algorithm (Collaborative Filtering)

Memory Based CF

The basic idea is to generate user-specific recommendations by finding specific patterns in a user’s series of behaviors. According to the implementation method, it can be divided into user-based collaborative filtering algorithms and item-based collaborative filtering algorithms.

Model Based CF

Model-based collaborative filtering algorithm

  • SVD algorithm (Singular Value Decomposition)
  • RBM algorithm (Restricted Boltzmann Machine)
  • Clustering algorithm (Clustering)
    • LSH (Local-sensitive Hashing)
    • K-means and its variants
    • LDA
    • Non-parametric Bayesian clustering
  • Association Rule
  • Classification algorithm (Classifiers)
    • Logical Regression
    • Bayesian Network
    • Support Vector Machine
    • Decision Tree
    • …

These models can be used for both CF algorithms and CB algorithms.

Advantages: Widely used, can be combined with other algorithms to improve accuracy

Disadvantages: Requires a relevant training set, may overfit

Advantages and Disadvantages of CF Algorithms

Input: Only depends on habitual data (such as ratings, purchases, downloads, and other user preference behaviors)

Advantages

  • Requires minimal knowledge
  • Good enough in most cases

Disadvantages

  • Sparsity: User overlap rate is very low in large datasets (such as a huge collection of goods like Taobao)
  • Computability: The amount of computation increases when the number of users and items in the domain calculation increases
  • Cold start problem
  • Requires standardized products
  • Popularity Bias: Does not perform well when there is a long tail
  • Difficult to provide explanations for recommendation results

Content-based Filtering Recommendation Algorithm (Content-based Filtering)

Recommend items to users that are similar to the content of items they liked in the past (such as metadata, descriptions, topics, etc.).

Advantages and Disadvantages

Input: Only depends on the content/description of items and users

Advantages

  • No cold start problem
  • No need for habitual data
  • No popularity bias, can recommend items with rare characteristics
  • Can use user content characteristics to provide explanations

Disadvantages:

  • Item content must be machine-readable and meaningful
  • Easy to stereotype users
  • Difficult to have surprises, lack of diversity

  • Difficult to combine the characteristics of multiple items
  • Easy to overfit

Hybrid Recommendation Algorithm

Introduction: Comprehensively utilize the advantages of collaborative filtering recommendation algorithms and content-based recommendation algorithms while offsetting their respective shortcomings

Types:

  • Weighted
  • Switching
  • Mixed
  • Feature Combination
  • Case
  • Feature Augmentation
  • Meta-level

Input: Use the content characteristics and habitual data of users and items at the same time, and benefit from both input types at the same time

Advantages and Disadvantages

Advantages

Since collaborative filtering recommendation algorithms and content-based recommendation algorithms are used alone

  • No cold start problem
  • No popularity bias, can recommend items with rare characteristics
  • Can generate surprises and achieve diversity

Disadvantages

  • Requires a lot of work to get the right balance

Other Non-traditional Algorithms

Learning to Rank

Understand the recommendation problem as a ranking problem. The loss function can be established from the perspective of points, pairs, and lists.

Similarity

SimRank: Calculate the similarity matrix based on bipartite graph theory, and use regression or MAB methods to combine the similarity matrices of different levels (such as metadata, user behavior data) as the final similarity matrix.

Deep Learning

Can be used for collaborative filtering methods or content-based recommendation algorithms

Social and Trust-based Recommender

Can be combined with collaborative filtering algorithms to introduce social relationships in similarity calculation; can also be combined with machine learning methods to introduce social relationships in regularization functions.

Can use demographic methods for classification recommendations.

Page Optimization

Mainly consider the optimization between the organization of the page and the distribution of user attention in different positions on the screen.

Tensor Factorization & Factorization Machine

In multi-dimensional scenarios such as user-resource-tags, data can be represented by tensors (usually very sparse). Tensor methods are used to decompose the tensors into low-dimensional space, and then methods in low-dimensional space (such as the bipartite graph-based methods mentioned earlier) are used for recommendation.

MAB Explore/Exploit

Use the Bandit algorithm in the Multi-armed bandit problem to solve the Explore/Exploit problem and the selection problem of different recommendation strategies.

Algorithms Used in the Industry

Alibaba - Graph Embedding

Main idea: Use dimensionality reduction methods to solve the problem of sparse matrices and large computational volume that would be encountered when directly applying CF algorithms in the case of a huge number of items.

Alibaba - Real-time Recommendation Algorithm SWING Based on Graph Structure

Main idea: Solve the problem of weak denoising ability of traditional point-based similarity algorithms through the SWING algorithm, and this method is more sensitive to changes in network structure.

Related Conferences

For the field of recommender systems, there are not many directly related conferences. However, since recommender systems involve knowledge of data mining, machine learning, etc., and recommender systems are an important application of data mining and machine learning, and recommender systems are also in the category of artificial intelligence when they are closer to larger fields, many scholars who do recommendation also turn their attention to conferences in data mining, machine learning, and artificial intelligence. Therefore, if you want to pay attention to the frontier of recommender systems, we need to pay attention not only to the annual conference of recommender systems, but also to other conferences related to recommendations.
