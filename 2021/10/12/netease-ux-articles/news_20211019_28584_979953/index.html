What We Talk About When We Talk About Map Data Visualization
12 Oct, 2021

Machine Translated from Chinese to English
Original article written by ThunderFire UX (NetEase)

Cognitive science research shows that illustrations can help people understand knowledge faster and retain it for longer periods. In the era of big data, we have also reached a consensus: data visualization can help us gain insight into the meaning of data. Map data visualization can express the spatial relationship between data in the most intuitive form, and has capabilities that no other method has. So what is map data visualization? What are its application scenarios? What are the implementation schemes on the Web? This article will discuss these one by one, and at the end of the article, some tools for map visualization will also be introduced.

What is Map Visualization and Its Application Scenarios

Map data visualization does not have a very official definition. It can be summarized as: Map data visualization is a data visualization method that visually presents data with geographical characteristics or analysis results on a geographic information map, which makes it easier for users to understand data patterns. The definition in words is not intuitive, and the following examples of application scenarios will make the definition easier to understand.

Starting from our surroundings, querying scenic spots and routes through map software (web pages) is a kind of map visualization, which visually marks locations, paths, road conditions and other information on real geographic information.

 

From a higher perspective, statistical analysis of earthquakes around the world over a period of time, the following figure marks the earthquake points on the world map, which can visually show the seismic zone.

 

In business activities, map visualization often appears in various reports to clearly show the distribution of data in geographic space.

 

In addition to real-world, national and other geographic information, other data visualizations related to spatial location can also be called map data visualization. For example, data visualization on a game map to show the distribution of birth points, character movement trajectories, and so on.

 

Through the above examples, the rules of map visualization can be summarized:

  1. Map visualization uses geographic information as the base map, and uses points, lines, and surfaces to represent data on the upper layer.
  2. Map visualization is usually used to represent the distribution and flow of data in geographic space.

Related Technologies for Map Data Visualization on the Web

Since the map uses geographic information as the base map and then adds data display of points, lines, and surfaces, the discussion of technical solutions can also be divided into two steps: geographic information base map rendering, and data layer visualization. The visualization of the data layer is basically the same as other data visualization methods, using canvas, svg or their upper-level technologies, but it is necessary to choose various projection functions to map the position to the base map. Therefore, this article will focus on the visualization technology of geographic information base maps.

In many systems, the geographic information base map does not actually render a map, but merely pastes a picture. The upper-level data is displayed through absolute or relative positioning, which is a visual deception, but it is effective and can save resources.

It can be seen that the base map in the report map mentioned earlier is composed of a series of polygons, which is a simple base map rendering method. Both Canvas and SVG in Web applications can render polygons (including curved edges).

Below is a visualization of an aircraft seat map. The base map is obtained through the graphic elements of SVG, and the data layer is filled with colors for the graphic elements.

 

The polygon description of maps of Chinese provinces, world countries, and other maps that we are familiar with has a unified format: GeoJSON (GeoJSON is a format for encoding various geographic data structures, a geographic spatial information data exchange format based on Javascript object notation). TopoJSON and other data formats have also been derived later. The coordinate points in GeoJSON are composed of longitude and latitude, so each piece of data in the data layer should also contain longitude, latitude, and numerical values. Some mature visualization libraries do not seem to require users to provide longitude and latitude, but actually complete the calculation internally. For example, in the following example from the Echarts official website, passing in the name of the area can complete the drawing of the data layer.

 

 

Similar to the earthquake distribution map in the previous section, maps that contain terrain information (need to render different heights), or as shown in the figure below, need to display spheres, need to use WebGL technology (WebGL is a JavaScript API, with a very consistent API with OpenGL ES 2.0, used for Web 3D rendering) for drawing. Mature 3D drawing libraries like Three.js are upper-level encapsulations based on WebGL. In actual use, choosing these mature underlying libraries can improve development efficiency.

 

The example of map navigation mentioned at the beginning of the article uses a high-precision base map, which is a completely different drawing technique. They may use a vector map solution, or a tile map (the tile map pyramid model is a multi-resolution hierarchical model. From the bottom to the top of the tile pyramid, the resolution becomes lower and lower, but the geographic range represented remains unchanged). When the map in the viewport is loaded in batches of rectangles, it uses the tile map technology solution. The accuracy of this type of map is much higher than that of polygon maps, and it can basically be accurate to buildings and shops. It is naturally superior to polygon maps in zooming interaction, so it is used for map data visualization with high precision requirements. The data layer also uses longitude and latitude to identify the data location. Now map service providers such as Mapbox, AutoNavi Maps, Tianditu, etc. all provide SDK for JavaScript. Mature chart libraries also encapsulate the calls of these SDKs, which are relatively convenient to use. Offline tile data can also be found online for developers to build offline map services. However, the development difficulty of this type of base map is higher than that of polygon maps, and there are different regulations on the number of calls and commercial access.

Map Data Visualization Tools

Excel

Excel provides map drawing functions, and Microsoft Power Map for Excel needs to be introduced. This is suitable for people who are familiar with Excel to perform static map data visualization. The usage methods of different Excel versions are not exactly the same.

 

Echarts

Echarts is a mature visualization library of js, which not only contains a small amount of 3D functions, but also uniformly provides high-precision base map API, which is suitable for most map data visualizations. Many BI platforms are implemented based on Echarts, but using Echarts requires users to have some programming skills. Programming languages such as Python and Matlab also have corresponding map visualization libraries.

 

Mapbox

Mapbox provides high-precision base map services and rich data layer drawing APIs, with highly free base map style customization, but as a lower-level map tool than Echarts, it requires higher programming skills.

 

Commercial BI such as NetEase Youdata

Many commercial platforms provide map data visualization functions. Their functions are rich and do not require users to program, but they basically require registration and fees.

 

Conclusion

Different data visualization methods are suitable for different information expressions. Map data visualization is suitable for expressing data distribution and flow related to geographic information. Map data visualization can be divided into two steps: base map drawing and data map drawing. Developers need to choose different technical solutions according to different needs scenarios. Data visualization has always been a method, and our real goal is to more clearly represent the information in the data through visualization.
