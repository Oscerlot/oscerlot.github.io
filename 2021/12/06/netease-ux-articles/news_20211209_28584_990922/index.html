I. Background

SVG is an image file format, the full name is Scalable Vector Graphics. It is a text file based on XML syntax. Most other image formats are based on pixel processing, so SVG does not distort like other image formats when enlarged.

Compared with PNG and JPEG images, SVG has a smaller size, stronger compressibility, and is scalable. It can be enlarged without losing image quality. It has an open standard and can be used to represent graphics, filters, animations, etc. It also has a very long history. It became a W3C standard in 2003, and mainstream browsers support SVG.

In practical work, SVG is very suitable for static image display, high-fidelity document viewing and printing, and other scenarios. Since SVG is described based on XML syntax, it is convenient to change the XML description, such as Path, Line, color, size, etc., to achieve the effect of changing SVG icons. Designers only need to produce one SVG image, which can be changed in size, color, animation, etc., and used in multiple places on the page.

Based on these drawbacks, we would prefer to have an SVG component library with unified standards, reusability, independence from third-party websites, support for on-demand loading, and easier access, use, maintenance, and expansion.

III. Solution Research

SVG Sprite

Webpack svg-sprite-loader supports merging SVG files into an SVG sprite and loading them into an element with display: none. When using it, pass in the id of the icon to render the icon. This solution supports on-demand loading and packaging.

SVG To React

Webpack svg-react-loader supports directly requiring an SVG image. Users only need to simply configure webpack to use it directly.

Convert SVG to React Component

If the SVG Sprite and SVG To React solutions are placed in an independent component library, the business needs to configure the loader to process the files of node_modules. For these two methods, there are shortcomings in the ease of use of the component library.

Therefore, consider converting each independent SVG in the component library into a pure React component, rendering and returning the XML of the SVG, and passing in Props to change the SVG.

V. Summary

  • svg is an image file format described based on xml syntax. The biggest advantage is that it does not distort when enlarged, and it is convenient to modify the size and color value. It is very suitable for static image display in actual work;
  • Companies often have their own common SVG Icons. Precipitating these SVG Icons into a component library can facilitate real-time access by designers, product managers, and developers;
  • The idea of building an SVG library is mainly reflected in converting SVG into an independent React Component, and then outputting the corresponding esm and cjs formats of the React Component;
  • The public SVG library can simplify the svg usage of icon font, and developers can easily change the svg; it can also unify and standardize the use and style of Icons in various products.
